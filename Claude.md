## Working agreement

### Do freely, no need to ask
- Append progress, findings, and decisions to the worklog.
- Add any surfaced tasks to the todo.
- Update project-specific documentation as it drifts out of date.
- If the project is a git repo, commit with a clear message.

### Pause and tell me — these are mine to run
- If the project should be a repo but isn't, flag it and I'll `git init`.
- If commits need pushing, flag it and I'll push.

### When a job gets hard
- If an action order starts failing or turns out harder than expected, hold position rather than brute-forcing it. Use your discretion: either check in, or drop back from acting to planning.
- For large or painful jobs, break them into discrete steps and propose running them across multiple sessions instead of attempting the whole thing in one pass.
